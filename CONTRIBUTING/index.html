    4. Be sure to test your code

    ‚úçÔ∏è Developer Certificate of Origin (DCO)

    Thank you for your contribution! To ensure transparency and legal clarity in the project, we use the Developer Certificate of Origin (DCO).

    üìú What is DCO?

    The DCO is a simple statement that you have the right to submit the code you‚Äôre contributing and that you agree to the project‚Äôs license. This is especially important for open source projects.

    Full DCO text: https://developercertificate.org

    ‚úÖ What do you need to do?

    Every commit must be signed. You can do this by adding the -s flag when creating a commit:

    git commit -s -m "fix: fixed error handling"

    Example of a commit signature:

    Signed-off-by: John Doe <john@example.com>

    üõ† Configure your name and email

    Make sure Git is configured with your name and email:

    git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"

    üîÅ How to re-sign commits

    If you forgot to sign a commit:

    One commit

    git commit --amend -s
git push --force

    Multiple commits

    git rebase -i HEAD~N  # Replace N with the number of recent commits

    Change pick to edit for the commits you want to sign, and for each:

    git commit --amend -s
git rebase --continue

    After finishing the rebase:

    git push --force

    ‚ö†Ô∏è Warning: use --force with caution, especially when working in a team.
